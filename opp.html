①
ความแตกต่างหลัก: Inheritance vs Encapsulation
การสืบทอดคุณสมบัติ vs การห่อหุ้มข้อมูล
🔗
Inheritance
การสืบทอดคุณสมบัติ
▸
Class ลูก (Child) รับ attribute และ method ทั้งหมดจาก Class แม่ (Parent) มาใช้ได้ทันที
▸
ใช้ keyword extends ในการสืบทอด
▸
ลดการเขียน code ซ้ำ (Code Reuse) — เขียนครั้งเดียวใช้ได้หลาย class
▸
สร้างความสัมพันธ์แบบ "is-a" เช่น Dog is-a Animal
▸
จุดเน้น: โครงสร้าง ของ class / ลำดับชั้น (Hierarchy)
VS
🔒
Encapsulation
การห่อหุ้มข้อมูล
▸
ซ่อน (hide) ข้อมูลภายใน class ด้วย private
▸
ภายนอกเข้าถึงได้ผ่าน Getter / Setter เท่านั้น ไม่แตะโดยตรง
▸
ป้องกัน data ถูกแก้ไขผิดพลาดจากภายนอก class
▸
ใส่ validation logic ใน Setter ได้ เช่น ห้ามอายุติดลบ
▸
จุดเน้น: ความปลอดภัย ของข้อมูล / การควบคุมการเข้าถึง
ตัวอย่าง: Inheritance + Encapsulation ในไฟล์เดียวกัน
class Animal { // Parent Class
  protected String name; // protected = ลูก access ได้
  public void speak() { System.out.println("..."); }
}

class Dog extends Animal { // ← Inheritance: Dog สืบทอดจาก Animal
  private String breed; // ← Encapsulation: ซ่อน breed ไว้ใน class

  public String getBreed() { return breed; } // Getter
  public void setBreed(String b) { this.breed = b; } // Setter

  @Override
  public void speak() { System.out.println("Woof!"); } // Override method
}
②
หลักการทำงานของการพ้องรูป (Polymorphism)
Method เดียวกัน — หลาย Class — ผลลัพธ์ต่างกัน
🐾 Animal — speak()
🐕 Dog
speak()
"Woof! 🐕"
🐱 Cat
speak()
"Meow! 🐱"
🐄 Cow
speak()
"Moo! 🐄"
✅ Method Overriding = Runtime Polymorphism — ทำงานตอน Run
✅ Method Overloading = Compile-time Polymorphism — ทำงานตอน Compile
ตัวอย่าง: ใช้ Polymorphism เรียก method ผ่าน array
Animal[] animals = { new Dog(), new Cat(), new Cow() };

for (Animal a : animals) {
  a.speak(); // เรียก method เดิมชื่อเดียวกัน
}
// Output: "Woof!" → "Meow!" → "Moo!"
// Java เลือก method ที่ถูกต้องให้อัตโนมัติ ณ Runtime!
③
บทบาทของ Getter และ Setter
ประตูเข้า-ออกข้อมูลของ Encapsulation
📤
Getter
getXxx() — อ่านค่า
👁
อ่านค่า private variableคืนค่ากลับให้ผู้เรียก ไม่แก้ไขข้อมูลใดๆ
🔒
ภายนอกเข้าถึงได้โดยไม่แตะ field โดยตรงปลอดภัยกว่า public variable
🔄
เปลี่ยน logic ภายในได้โดยไม่กระทบ code ที่เรียกใช้ภายนอก
📥
Setter
setXxx() — เขียนค่า
✏️
เปลี่ยนค่า private variableรับ parameter แล้ว assign ให้ field
✅
ใส่ Validation ได้เช่น ห้าม age < 0 หรือ name ห้ามเป็น null
🛡
ป้องกัน data ผิดรูปแบบก่อนที่ข้อมูลจะเข้าสู่ระบบ
ตัวอย่าง Getter & Setter พร้อม Validation
class Person {
  private String name; // private = ซ่อน, เข้าถึงตรงๆ ไม่ได้
  private int age;

  // === GETTER ===
  public String getName() { return name; }
  public int getAge() { return age; }

  // === SETTER + VALIDATION ===
  public void setName(String name) {
    if (name != null && !name.isEmpty()) this.name = name;
  }
  public void setAge(int age) {
    if (age > 0) this.age = age; // ห้ามอายุติดลบ!
  }
}

// การใช้งาน
Person p = new Person();
p.setName("สมชาย"); // ✅ ผ่าน Setter
p.setAge(-5); // ❌ ถูกบล็อก! อายุติดลบไม่ได้
System.out.println(p.getName()); // ✅ ผ่าน Getter

API ในรูปแบบ Client–Server
การสื่อสารระหว่าง 2 ฝั่งผ่าน HTTP Request/Response
💻
CLIENT
Browser, Mobile App
หรือ Program ที่ต้องการข้อมูล

หน้าที่: ส่ง Request
และ แสดงผล Response
→ REQUEST →
HTTP / HTTPS
JSON / XML
← RESPONSE ←
🖥
SERVER
Backend Program + Database
ที่ให้บริการข้อมูล

หน้าที่: รับ Request,
ค้นหา DB, ส่งผลกลับ
1
Client ส่ง Request
ระบุ URL endpoint, HTTP method, และ data (ถ้ามี)
2
Server รับและประมวลผล
ตรวจสอบสิทธิ์, ค้นหา DB, เตรียมข้อมูล
3
Server ส่ง Response
ส่งกลับในรูป JSON พร้อม HTTP Status Code
4
Client แสดงผล
นำ JSON ไป parse และแสดงผลบน UI
GET
ดึงข้อมูล
ไม่แก้ไขอะไร
POST
สร้างข้อมูลใหม่
ส่ง body ไปด้วย
PUT
แก้ไขข้อมูล
ส่งข้อมูลใหม่ทั้งหมด
DELETE
ลบข้อมูล
ระบุ ID ที่ต้องการลบ
ตัวอย่าง: เรียก API ด้วย Java (GET request)
// Client ส่ง HTTP GET ไปยัง Server
URL url = new URL("https://api.example.com/users/1");
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setRequestMethod("GET");

// อ่าน Response จาก Server
BufferedReader reader = new BufferedReader(
  new InputStreamReader(conn.getInputStream())
);
String jsonResponse = reader.readLine();
// jsonResponse = { "id": 1, "name": "สมชาย", "age": 25 }

แนวคิด OOP สำหรับการเรียกใช้งาน API
ออกแบบโปรแกรมเชิงวัตถุให้รองรับ API อย่างมีโครงสร้าง
🎮
Controller / Main Class
รับ input จาก User หรือ UI → เรียก Service → แสดงผล
แยก logic การทำงานออกจากส่วนแสดงผล
App.main() → userService.getUsers() → display()
↕
🌐
Service Class (API Layer)
จัดการ HTTP Request/Response, parse JSON กลับมาเป็น Object
แยก network logic ออกจาก business logic
UserService.fetchUsers() → HTTP GET → return User[]
↕
📦
Model Class (Data)
แทนโครงสร้างข้อมูลที่รับจาก API เช่น User, Product, Order
มี attribute + Getter/Setter ครบ
class User { private int id; private String name; ... }
ตัวอย่าง OOP + API ครบทุก Layer
// 📦 Model Layer
class User {
  private int id;
  private String name;
  public int getId() { return id; }
  public String getName() { return name; }
  public void setName(String n){ name=n; }
}

// 🌐 Service Layer
class UserService {
  public User[] fetchUsers() {
    String json = httpGet("/api/users");
    return parseJson(json, User[].class);
  }
}

// 🎮 Controller Layer
class App {
  public static void main(String[] args) {
    UserService svc = new UserService();
    User[] users = svc.fetchUsers();
    for(User u : users)
      System.out.println(u.getName());
  }
}
🎯 ประโยชน์ของ OOP + API
✦
Separation of Concerns — แต่ละ layer ทำหน้าที่ของตัวเอง ไม่ยุ่งกัน
✦
Reusability — UserService ใช้ซ้ำได้ทั้งระบบ ไม่ต้องเขียนใหม่
✦
Testability — Test ได้ทีละ layer ง่าย ต่อการ debug
✦
Maintainability — เพิ่ม feature ใหม่ โดยไม่กระทบ code เดิม
✦
Encapsulation — ซ่อน network detail ไว้ใน Service ภายนอกไม่รู้
